# Dolphin ID v0.1.0

Release date: 2026-06-01

## Summary

The v0.1.0 MVP release establishes the first supported Dolphin ID authentication flow for wallet-based sign-in. It includes the shared adapter contracts, server nonce/JWT verification core, EVM and Sui adapters, headless React hooks, default wallet UI, a Next.js example with E2E coverage, and MVP security/documentation guidance.

## Supported chains

- EVM wallets using SIWE-compatible signing.
- Sui wallets using personal-message signing.

## Package versions

All MVP workspace packages and examples in this release branch are versioned as `0.1.0`:

- `@dolphin-id/core`
- `@dolphin-id/server`
- `@dolphin-id/react`
- `@dolphin-id/ui`
- `@dolphin-id/adapter-evm`
- `@dolphin-id/adapter-sui`
- `examples/basic`
- `examples/next`

## Changelog

- Added core adapter contracts and package boundaries.
- Added a typed error, event, and state model shared by adapters and UI.
- Added nonce issuance and JWT session verification for Node server usage.
- Added EVM SIWE and Sui personal-message login slices.
- Added headless React provider/hooks and default connect-wallet UI.
- Added a Next.js example with E2E sign-in verification.
- Added MVP security hardening, API reference, getting-started, and troubleshooting docs.

## Migration notes

- This is the first tagged release, so there is no prior public Dolphin ID version to migrate from.
- Consumers should pin all Dolphin ID packages to `0.1.0` together; mixing `0.1.0` with later prerelease packages is not supported.
- Server deployments must configure nonce storage, JWT signing secrets, allowed origins, and chain-specific verification settings before accepting production traffic.

## Known limits

- Supported wallet families are limited to EVM and Sui.
- Refresh tokens, force logout, hosted session services, and multi-wallet identity linking are not part of v0.1.0.
- Mobile WalletConnect, Solana, Bitcoin, Aptos, CLI scaffolding, and non-TypeScript SDKs are planned for later releases.

## Upgrade path

- For new apps, start from `examples/next` and replace the demo server settings with deployment-specific configuration.
- For existing prototypes, align all Dolphin ID packages to `0.1.0`, update imports to the documented package boundaries, and verify the EVM/Sui sign-in flow with the provided E2E pattern.
- Upgrade to v1.0.0 when stable multi-chain adapters, refresh/logout controls, hosted nonce/session flows, CLI scaffolding, and Go/Rust/Python server SDK parity are required.
