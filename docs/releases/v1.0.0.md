# Dolphin ID v1.0.0

Release date: 2026-06-01

## Summary

The v1.0.0 stable release completes the production-ready Dolphin ID plan: core wallet authentication packages, EVM/Sui/Solana/Bitcoin/Aptos adapters, React and UI packages, server auth primitives, refresh/logout controls, Express/Fastify middleware, the hosted nonce/session service, CLI scaffolding, docs site, third-party adapter specification, and Go/Rust/Python server SDK parity.

## Stable version scope

- npm workspace packages: `1.0.0`
- Rust SDK crate: `dolphin-id-server` `1.0.0`
- Python SDK package: `dolphin-id-server` `1.0.0`
- Go SDK module: versioned by the `v1.0.0` git tag

## Supported chains and adapters

- EVM SIWE, including WalletConnect-compatible provider support and mobile deep-link helpers.
- Sui personal-message signing.
- Solana SIWS.
- Bitcoin P2PKH/secp256k1 SIWX.
- Aptos Ed25519 SIWX.
- Third-party adapters implementing the documented v1.0 adapter specification.

## Changelog

- Added multi-wallet identity linking with primary account selection and sensitive-operation policy checks.
- Added refresh token rotation, refreshable React session state, logout, and server-side session invalidation.
- Added Express and Fastify middleware for authenticated routes.
- Added Solana, Bitcoin, and Aptos adapters with matching server verification helpers.
- Added WalletConnect/mobile EVM deep-link support.
- Added the hosted nonce/session service with project API keys, domain checks, quotas, billing hooks, and audit logs.
- Added CLI app scaffolding for Next.js EVM/Sui starter apps.
- Added Go, Rust, and Python server SDKs with shared verification fixtures.
- Added the docs site, adapter specification, adapter test fixtures, migration guidance, and security posture documentation.
- Completed v1.0 security audit remediation for hosted scoping, production defaults, project domain/quota checks, and hosted failure audit trails.

## Security audit

The v1.0 security summary is published in `docs/security-audit.md`. It covers adapter trust boundaries, nonce/session lifecycle, refresh/logout invalidation, hosted project isolation, API key handling, quota enforcement, and the remediation status for critical/high findings.

## Migration guidance

- Upgrade all Dolphin ID npm packages together to `1.0.0`.
- Replace draft docs references with the v1.0 stable docs site content.
- If moving from the v0.1 MVP, review the new refresh/logout endpoints, middleware helpers, and multi-wallet identity model before enabling production sessions.
- For hosted auth, configure project API keys, allowed domains, quotas, and production secrets before accepting traffic.
- For non-TypeScript servers, use the Go module at tag `v1.0.0` or the `1.0.0` Rust/Python packages and validate fixture parity in CI.

## Known limits

- Hosted billing hooks are integration points; billing provider settlement remains application-owned.
- Third-party adapters must pass the published adapter fixture expectations before being treated as production-supported.
- Go modules are versioned through git tags, so consumers should import the module at `v1.0.0`.
